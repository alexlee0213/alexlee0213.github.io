## 개요

OpenClaw은 AI가 컴퓨터를 직접 조작하는 자율 에이전트로, 2026년 2~4월 사이 138개 이상의 보안 결함(CVE)이 보고되었다. Microsoft, Cisco, Kaspersky 등 글로벌 보안 기관이 일제히 경고를 발표했으며, 중국 정부는 정부기관·국영기업에서의 사용을 제한했다.

본 가이드는 기술 상세 보고서(종합보고서)를 기반으로, 기술적 배경 없이도 핵심 위험과 대응 방법을 이해할 수 있도록 요약한 문서이다.

---

## 핵심 위험 요약

| 위험 유형 | 설명 | 심각도 |
|-----------|------|--------|
| 비밀번호/키 노출 | AI가 에러 로그에 민감 정보를 평문 기록 | 높음 |
| 악성 플러그인 | 공식 마켓(ClawHub)의 12~20%가 악성 | 높음 |
| 웹 해킹(ClawJacked) | 악성 웹사이트 방문만으로 AI 탈취 가능 | 중간 |
| 대규모 데이터 유출 | API 토큰 150만개 + 이메일 35,000개 실제 유출 | 매우 높음 |
| 프롬프트 조작 | 외부에서 AI에게 몰래 명령 주입 가능 | 중간 |

## 설치 환경별 안전도

| 설치 위치 | 안전도 | 설명 |
|-----------|--------|------|
| 별도 리눅스 서버 + Docker | ★★★★★ | 가장 추천. 이중 격리 보호 |
| 별도 리눅스 서버 (직접 설치) | ★★★★ | 별도 컴퓨터라 좋지만 보안 설정 직접 필요 |
| 메인 컴퓨터 + Docker | ★★★ | 어느 정도 보호되나 메인 PC에 영향 가능 |
| Windows WSL2 | ★★ | 격리처럼 보이지만 실제로는 Windows와 맞붙어 있음 |
| **메인 컴퓨터에 직접 설치** | **★** | **절대 비추천** |

## 필수 보안 규칙

**하지 마세요:**
1. 업무용 PC나 개인 메인 컴퓨터에 직접 설치
2. 검증 없이 Skill(확장 기능) 설치
3. 인증 기능을 끄거나 무시
4. 업데이트 없이 방치
5. AI에게 무제한 권한 부여

**반드시 하세요:**
1. 별도 컴퓨터(VM 또는 리눅스 서버)에서 사용
2. 항상 최신 버전 유지 (v2026.4.7-1 이상)
3. 게이트웨이 인증 활성화 확인 (토큰 자동 생성)
4. AI의 중요한 행동에 "사람 승인" 활성화
5. 관리자/IT팀에게 보안 설정 요청
