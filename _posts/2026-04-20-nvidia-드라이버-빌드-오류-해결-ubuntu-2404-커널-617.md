## 환경

- OS: Ubuntu 24.04 LTS (HWE 커널 사용)
- 커널: 6.17.0-20-generic
- GPU: NVIDIA GeForce GTX 1660 SUPER (6GB)
- 기존 드라이버: nvidia-dkms-575 (575.64.03)

## 증상

커널이 6.17.0-20-generic으로 업데이트된 후, NVIDIA DKMS 모듈 빌드가 실패하며 다음과 같은 연쇄 오류가 발생했다.

```
Error! Bad return status for module build on kernel: 6.17.0-20-generic (x86_64)
Consult /var/lib/dkms/nvidia/575.64.03/build/make.log for more information.
dkms autoinstall on 6.17.0-20-generic/x86_64 failed for nvidia(10)
```

이로 인해 `linux-headers-6.17.0-20-generic`, `linux-headers-generic-hwe-24.04`, `linux-generic-hwe-24.04` 패키지가 모두 미설정(not configured) 상태로 남았다.

## 원인 분석

커널 6.16 이상부터 커널 내부 API가 변경되어(`dma_buf_attachment_is_dynamic` 등), **NVIDIA 드라이버 580 미만 버전은 컴파일 자체가 불가능**하다. 575.64.03 드라이버는 커널 6.17을 지원하지 않는다.

### 연쇄 오류 흐름

1. DKMS nvidia/575.64.03 모듈 빌드 실패 (`bad exit status: 2`)
2. `dkms autoinstall` 실패 (exit code 10)
3. `/etc/kernel/header_postinst.d/dkms` 스크립트 실패 (exit code 11)
4. `linux-headers-6.17.0-20-generic` 패키지 post-install 실패 → 미설정 상태
5. 의존 패키지 `linux-headers-generic-hwe-24.04` → 설정 불가
6. 의존 패키지 `linux-generic-hwe-24.04` → 설정 불가

## 해결 과정

### 1단계: NVIDIA 드라이버 580으로 업그레이드

```bash
sudo apt update
sudo apt install nvidia-dkms-580
```

이 경우에는 이미 580.126.09가 설치되어 있었고, 구버전 575가 DKMS에 잔존하는 상태였다.

### 2단계: DKMS 상태 확인 및 구버전 정리

```bash
# DKMS 등록 상태 확인
dkms status

# 575가 남아 있다면 제거
sudo dkms remove nvidia/575.64.03 --all
sudo apt purge nvidia-dkms-575 nvidia-kernel-source-575
```

### 3단계: 미설정 패키지 복구

```bash
sudo dpkg --configure -a
sudo apt --fix-broken install
```

### 4단계: 재부팅 후 확인

```bash
# 커널 확인
uname -r

# 드라이버 동작 확인
nvidia-smi
```

## 최종 결과

```
NVIDIA-SMI 580.126.09    Driver Version: 580.126.09    CUDA Version: 13.0
GPU: NVIDIA GeForce GTX 1660 SUPER
```

드라이버 580.126.09가 커널 6.17.0-20-generic에서 정상 동작하는 것을 확인했다.
