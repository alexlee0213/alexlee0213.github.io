## 개요

Claude는 7개의 실행 환경(claude.ai, Desktop, Cowork, Code, API, Slack, Office 플러그인)을 제공하며, 각 환경은 독립적이고 격리된 샌드박스 구조를 갖는다. 본 문서는 각 환경의 보안 격리 방식, 핵심 기능, 장단점, 그리고 사용 시나리오별 추천 환경을 비교 분석한 가이드이다.

---

## 환경별 핵심 특징 요약

### claude.ai (웹 인터페이스)

브라우저만으로 접속 가능한 가장 대표적인 환경. 아티팩트(Artifacts)로 코드·문서·다이어그램을 시각적으로 확인하고, 분석 도구(Analysis Tool)로 서버 측 Python 샌드박스에서 코드를 실행할 수 있다. 프로젝트, 웹 검색, 원격 MCP 통합, 리서치 기능을 지원하며, 코드 실행은 격리된 서버 측 컨테이너에서 수행되어 사용자 시스템과 완전 분리된다.

### Claude Desktop (네이티브 앱)

macOS/Windows용 앱으로 claude.ai의 핵심 기능에 더해 Desktop Extensions를 통한 로컬 MCP 서버 연결, Claude Code 통합을 제공한다. 서버 측 처리 기반으로 로컬 파일시스템에 직접 접근하지 않는다(Cowork 모드 제외).

### Claude Cowork (에이전트 모드)

Desktop 앱 내 에이전트 모드로, 격리된 VM에서 실행되며 사용자가 선택한 폴더에 직접 읽기/쓰기가 가능하다. 샌드박스 Linux 셸에서 Python, Node.js, CLI 도구를 사용하고, 문서 생성(docx, pptx, xlsx, pdf) 전문 스킬이 내장되어 있다.

### Claude Code (터미널/IDE)

개발자용 에이전트 코딩 도구. OS 수준 샌드박스(Linux bubblewrap, macOS seatbelt)로 파일시스템과 네트워크를 격리한다. 현재 작업 디렉토리에만 접근 허용하며, Unix 도메인 소켓 기반 프록시로 네트워크를 제어한다. 웹 기반 실행 시 클라우드 컨테이너에서 동작한다.

### Claude API (개발자 플랫폼)

자체 애플리케이션에 Claude를 통합하는 프로그래밍 인터페이스. 샌드박스 구성은 전적으로 개발자 책임이며, Code Execution Tool과 Agent SDK를 통한 컨테이너 격리를 권장한다.

### Claude in Slack / Office 플러그인

Slack 워크스페이스나 Excel/Chrome 내에서 동작하며, 호스트 앱의 기존 권한 시스템을 준수한다. 로컬 파일 접근이나 코드 실행 기능은 없다.

## 보안 격리 수준

| 격리 요소 | claude.ai | Desktop/Cowork | Code | API | Slack/Office |
|-----------|-----------|----------------|------|-----|--------------|
| 프로세스 격리 | 서버 컨테이너 | 격리 VM | OS 샌드박스 | 개발자 책임 | 호스트 앱 내부 |
| 파일시스템 | 서버 측 처리 | 선택적 접근 | CWD 제한 | 제한 없음* | 미접근 |
| 네트워크 | API 서버 | 허용목록 | 프록시 | 개발자 설정 | 호스트 앱 의존 |
| 권한 제어 | 요금제 기반 | 자동/사용자 | 사용자 확인 | API 키 기반 | 관리자 통제 |

*API는 개발자가 Agent SDK 등을 통해 자체적으로 샌드박스를 구성해야 한다.

## 사용 시나리오별 추천 환경

| 시나리오 | 추천 환경 | 이유 |
|----------|-----------|------|
| 일반 질문/대화 | claude.ai | 설치 불필요, 브라우저에서 즉시 사용 |
| 데이터 분석/시각화 | claude.ai | Analysis Tool로 Python 실행 + 아티팩트 시각화 |
| 심층 리서치 | claude.ai | 리서치 기능으로 수백 개 소스 탐색 |
| 보고서/문서 작성 | Cowork | 로컬 파일 접근 + 문서 생성 스킬 내장 |
| 코드 개발/디버깅 | Code | OS 수준 샌드박스 + 전체 터미널 접근 |
| 애플리케이션 통합 | API | 최대 유연성 + 커스텀 샌드박스 구성 |
| 팀 협업 | Slack | 기존 워크플로우 통합 + 권한 자동 준수 |

## 핵심 보안 원칙

모든 Claude 실행 환경은 5계층 보안 구조를 공유한다:

1. **L1 모델 안전성** — Claude 모델 자체의 안전성 학습 및 가드레일
2. **L2 권한 제어** — 사용자 확인, API 키, 관리자 정책을 통한 접근 통제
3. **L3 프로세스 격리** — VM, 컨테이너, OS 샌드박스를 통한 실행 환경 격리
4. **L4 파일시스템** — 작업 디렉토리 제한, 선택적 폴더 접근, 민감 파일 보호
5. **L5 네트워크** — 프록시 기반 도메인 허용목록, 승인된 연결만 허용

각 환경은 완전히 독립적으로 작동하며, 환경 간 데이터 공유는 발생하지 않고, 세션 종료 시 임시 상태가 초기화된다.
