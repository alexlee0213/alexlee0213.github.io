[NomaDamas/k-skill](https://github.com/NomaDamas/k-skill)은 AI 에이전트(Claude, Codex 등)가 한국 공공 API와 서비스를 바로 사용할 수 있도록 만든 스킬 모음집이다. 설치부터 실제 사용, 프록시 서버 분석, 보안 검토, 내부 아키텍처 분석까지 진행한 기록을 정리한다.

---

## 1. 설치

설치 문서([docs/install.md](https://github.com/NomaDamas/k-skill/blob/main/docs/install.md))에 따라 전체 스킬을 한 번에 설치한다.

```bash
npx --yes skills add NomaDamas/k-skill --all -g
```

27개 스킬이 `~/.agents/skills/`에 글로벌 설치되었다. 설치 후 `k-skill-setup` 스킬을 실행하여 credential 파일(`~/.config/k-skill/secrets.env`)을 생성하고 런타임 환경(Node v22, Python 3.10, npm/npx)을 확인했다.

### 설치된 27개 스킬 목록

| 분류 | 스킬 |
|------|------|
| 생활/쇼핑 | coupang-product-search, daiso-product-search, olive-young-search, delivery-tracking, zipcode-search, used-car-price-search, blue-ribbon-nearby, kakao-bar-nearby |
| 교통/예매 | srt-booking, ktx-booking, seoul-subway-arrival, cheap-gas-nearby |
| 스포츠/게임 | kbo-results, kleague-results, lck-analytics, lotto-results |
| 금융 | toss-securities |
| 공공/법률 | korean-law-search, real-estate-search, household-waste-info, fine-dust-location, han-river-water-level, joseon-sillok-search |
| 유틸리티 | hwp, korean-spell-check, kakaotalk-mac, k-skill-setup |

### credential 설정

`~/.config/k-skill/secrets.env` (퍼미션 0600)에 아래 변수를 관리한다.

```
KSKILL_SRT_ID / KSKILL_SRT_PASSWORD    → SRT 예매
KSKILL_KTX_ID / KSKILL_KTX_PASSWORD    → KTX 예매
LAW_OC                                  → 한국 법령 검색 (로컬)
AIR_KOREA_OPEN_API_KEY                  → 미세먼지 (직접 호출 시)
KSKILL_PROXY_BASE_URL                   → 프록시 서버 URL
```

부동산, 쓰레기 배출, 주유소, 한강 수위 등은 기본 hosted proxy(`k-skill-proxy.nomadamas.org`)를 사용하므로 별도 API 키가 필요 없다.

## 2. 실제 사용 테스트

### 서울 지하철 실시간 도착정보

서대문역 광화문 방면 열차를 조회했다.

```bash
curl -fsS --get 'https://k-skill-proxy.nomadamas.org/v1/seoul-subway/arrival' \
  --data-urlencode 'stationName=서대문'
```

5호선 하행(광화문 방면) 마천행 열차가 약 2분 후, 하남검단산행이 약 8분 후 도착 예정이었다. 프록시가 서울 열린데이터 API 키를 서버 측에서 주입하고, 300초 TTL 캐시를 적용한다.

### 로또 최신 회차 당첨번호

```javascript
const lotto = require("k-lotto");
const round = await lotto.getLatestRound();    // 1218
const result = await lotto.getDetailResult(round);
```

제1218회(2026-04-04 추첨) 당첨번호: **3 · 28 · 31 · 32 · 42 · 45 + 보너스 25**. 1등 18명, 1인당 약 17억 1,448만 원. k-lotto 패키지는 동행복권(`dhlottery.co.kr`)을 직접 호출하며, 프록시를 거치지 않는다.

## 3. k-skill-proxy 서버 조사

### 운영 주체와 인프라

- **운영:** NomaDamas (Markr AI 후원 AI 오픈소스 해커 하우스)
- **서버:** Cloudflare Tunnel → Node.js + PM2 (port 4020)
- **리전:** Cloudflare 홍콩(HKG) 엣지에서 응답
- **배포:** main 브랜치 merge → 매시 cron 자동 배포

### 프록시하는 공공 API 8개

```
/v1/seoul-subway/arrival    → 서울 열린데이터 광장 (지하철)
/v1/fine-dust/report        → AirKorea (미세먼지)
/v1/han-river/water-level   → HRFCO (한강 수위)
/v1/opinet/around, /detail  → 한국석유공사 (주유소 가격)
/v1/real-estate/*           → 국토교통부 (부동산 실거래가)
/v1/korean-stock/*          → 한국거래소 (주식 시세)
/v1/household-waste/info    → 공공데이터포털 (쓰레기 배출)
/v1/korea-weather/forecast  → 기상청 (날씨, 미설정 상태)
```

### 보안과 기밀성

프록시를 통과하는 데이터는 역명, 지역명, 좌표, 종목코드 등 **조회 파라미터와 공공 데이터 응답**뿐이다. SRT/KTX 예매처럼 로그인이 필요한 스킬은 프록시를 거치지 않는다.

**리스크 요인:**
- 프록시 운영자가 서버 로그로 요청 패턴을 볼 수 있음
- Cloudflare + 프록시 서버가 복호화된 요청을 볼 수 있는 위치
- 현재 무인증 공개 정책 ("문제가 생기면 그때 방어 추가")

**완화 요인:**
- 공공 데이터만 취급 (개인정보/결제정보 미경유)
- read-only (write 작업 없음)
- self-host 가능 (자체 프록시로 완전 우회 가능)

## 4. 전체 아키텍처 — 스킬의 4가지 유형

k-skill의 핵심은 **스킬 자체가 코드가 아니라 자연어 명세서(SKILL.md)**라는 점이다. LLM이 이 문서를 읽고 워크플로우를 직접 실행한다.

### 유형 A: k-skill-proxy 경유형

```
사용자 질의 → LLM이 SKILL.md 해석 → curl로 프록시 호출
  → Cloudflare Tunnel → Node.js 서버 → upstream 공공 API
    → JSON 응답 → LLM이 자연어로 요약
```

해당: seoul-subway-arrival, fine-dust-location, han-river-water-level, cheap-gas-nearby, real-estate-search, household-waste-info, blue-ribbon-nearby

### 유형 B: npm 패키지 직접 실행형

```
사용자 질의 → LLM이 SKILL.md 해석 → npm install → node 스크립트 실행
  → 패키지가 대상 사이트 직접 호출 → JSON 결과 → LLM이 요약
```

해당: lotto-results (k-lotto), kbo-results (kbo-game), kleague-results, lck-analytics, coupang-product-search, daiso-product-search, olive-young-search 등

### 유형 C: 인증 필요 + CLI/Python형

```
사용자 질의 → LLM이 SKILL.md 해석 → credential 확보 (secrets.env)
  → CLI/Python 실행 (tossctl, SRTrain, korail2)
    → 인증 후 서비스 직접 호출 → 결과 → LLM이 요약
```

해당: srt-booking, ktx-booking, toss-securities, korean-law-search, kakaotalk-mac

### 유형 D: MCP 서버 경유형

```
사용자 질의 → LLM이 SKILL.md 해석 → MCP Streamable HTTP 세션 초기화
  → tools/call JSON-RPC → MCP 서버가 실제 서비스 호출 → 결과 → LLM이 요약
```

해당: coupang-product-search (HuggingFace Space coupang-mcp)

## 6. 새 스킬 추가 시 필요한 작업

### SKILL.md 필수 섹션

```
What this skill does → When to use → Prerequisites
→ Required environment variables → Inputs → Workflow → Done when
→ Failure modes → Notes
```

### 유형별 추가 작업

- **프록시 경유형:** 서버 코드에 라우트 추가 → upstream API 키 등록 → health에 반영 → main merge → 자동 배포
- **npm 패키지형:** 패키지 개발 → npm publish → SKILL.md에 설치/실행 코드 작성
- **인증형:** CLI/Python 패키지 준비 → secrets.env에 변수 추가 → k-skill-setup에 항목 추가
- **MCP형:** MCP 서버 준비 → SKILL.md에 초기화 + tools/call 패턴 작성

## 마무리

k-skill은 LLM이 SKILL.md라는 자연어 명세서를 해석해서 한국 공공 API를 호출하는 구조라는 점이 흥미롭다. 전통적인 SDK/라이브러리와 달리 프로그래밍 없이 문서만으로 기능을 정의하고, 프록시 서버로 API 키 관리를 중앙화한 설계가 인상적이다. 다만 프록시 서버의 무인증 공개 정책과 운영 투명성은 프로덕션 환경에서 검토가 필요한 부분이다.
