# NanoClaw + 구글 캘린더: 매일 아침 일정 브리핑 자동화하기

매일 아침 오늘의 일정을 텔레그램으로 받아보고 싶었다. 구글 캘린더 API를 쓰면 되지만, OAuth 토큰 관리가 번거롭고, 개인 계정 연동은 가급적 피하고 싶었다. 결론부터 말하면, **iCal 비공개 URL + NanoClaw 태스크 스크립트**로 인증 없이 깔끔하게 해결했다.

## 요구사항

- 구글 캘린더 **2~3개**의 오늘 일정을 매일 아침 8시에 텔레그램으로 브리핑
- 계정 OAuth 연동 **없이** 구현
- 반복 일정(매주/매월/매년) 지원
- 종일 일정, 시간 일정 모두 표시

## 방법 비교: 3가지 선택지

| 방법 | 인증 | 난이도 | 읽기/쓰기 | 실시간성 |
|------|------|--------|-----------|---------|
| **iCal 비공개 URL** | 없음 (URL만 있으면 됨) | 쉬움 | 읽기 전용 | 수분~수십분 지연 |
| Google API + 서비스 계정 | JSON 키 파일 | 보통 | 읽기+쓰기 | 실시간 |
| Google API + OAuth | 토큰 갱신 필요 | 복잡 | 읽기+쓰기 | 실시간 |

매일 아침 브리핑 용도로는 읽기 전용이면 충분하고, 수분 지연은 전혀 문제가 안 된다. **iCal URL이 압도적으로 간단하다.**

## Step 1: 구글 캘린더에서 iCal URL 가져오기

> PC 웹 브라우저에서만 가능하다. 모바일 앱에서는 이 설정이 보이지 않는다.

1. [calendar.google.com](https://calendar.google.com) 접속
2. 좌측 상단 ⚙️ → **설정**
3. 좌측 사이드바에서 **"내 캘린더의 설정"** → 원하는 캘린더 클릭
4. 우측 패널을 아래로 스크롤 → **"캘린더 통합"** 섹션
5. **"비공개 주소 (iCal 형식)"** 의 URL을 복사

URL은 이런 형태다:

```
https://calendar.google.com/calendar/ical/{이메일}/private-{해시}/basic.ics
```

### 확인 방법

복사한 URL을 브라우저 주소창에 붙여넣으면 `.ics` 파일이 다운로드된다. 파일이 받아지면 성공.

### 보안 주의

- 이 URL을 아는 사람은 **누구나** 일정을 볼 수 있다
- 절대 공개 저장소나 블로그에 원본 URL을 올리지 말 것
- URL이 유출되면: 같은 설정 페이지에서 **"비공개 URL 재설정"** 클릭 → 기존 URL 무효화

### Google Workspace 계정인 경우

회사/학교 계정이면 관리자가 iCal 공유를 비활성화했을 수 있다. 이 경우 관리자에게 "캘린더 공유 설정에서 iCal 비공개 URL 허용"을 요청하거나, 서비스 계정 방식으로 전환해야 한다.

## Step 2: NanoClaw 태스크 스크립트 작성

NanoClaw의 태스크 시스템은 두 단계로 동작한다:

1. **스크립트 단계**: bash 스크립트가 먼저 실행되어 데이터를 수집
2. **에이전트 단계**: 스크립트가 `wakeAgent: true`를 반환하면, LLM 에이전트가 깨어나서 데이터를 가공·전송

이 구조 덕분에 **iCal 파싱 같은 무거운 작업은 스크립트가 처리**하고, 에이전트는 예쁘게 포맷팅해서 보내는 역할만 한다. API 크레딧도 절약된다.

### ICS 파싱 스크립트

핵심은 `.ics` 파일을 fetch해서 오늘 일정만 필터링하는 것이다. 주의할 점이 몇 가지 있다:

#### UTC → KST 변환

구글 캘린더의 ICS 파일에서 시간은 두 가지 형태로 나온다:

```
# UTC (Z 접미사) — 변환 필요
DTSTART:20260415T063000Z      → KST 15:30

# TZID 명시 — 이미 해당 타임존
DTSTART;TZID=Asia/Seoul:20260415T153000  → KST 15:30
```

Z로 끝나면 UTC이므로 +9시간 해야 한다. 이걸 놓치면 시간이 9시간 밀린다.

#### 종일 일정

```
DTSTART;VALUE=DATE:20260415
DTEND;VALUE=DATE:20260416
```

시간 없이 날짜만 있으면 종일 일정이다. DTEND가 다음 날인 것에 주의 — 이건 ICS 표준의 관례다 (종료일은 exclusive).

#### 반복 일정 (RRULE)

```
RRULE:FREQ=WEEKLY;BYDAY=MO,WE,FR
RRULE:FREQ=YEARLY
RRULE:FREQ=MONTHLY;INTERVAL=2
```

반복 일정은 `DTSTART`가 과거 날짜이고 `RRULE`로 반복 패턴이 정의된다. 오늘이 해당 패턴에 맞는지 계산해야 한다.

위 세 가지 함정(UTC→KST, 종일 일정 DTEND exclusive, RRULE 처리)만 제대로 다루면 실제 구현은 100줄 남짓이다. NanoClaw에게 "이 함정들을 피해서 ICS 파싱 스크립트를 짜줘"라고 자연어로 지시하면 된다. 전체 코드를 여기 옮겨 적는 대신, 스크립트가 에이전트에게 넘겨주는 **데이터 규격**만 합의해 두면 충분하다:

```javascript
// 스크립트의 마지막 한 줄 — 에이전트 기동 + 데이터 전달
console.log(JSON.stringify({ wakeAgent: true, data: result }));
```

### 스크립트가 에이전트에게 넘기는 데이터

스크립트는 캘린더별로 오늘 일정을 모아 아래 모양의 JSON을 `stdout`으로 내보낸다. 에이전트는 이 데이터를 받아 사람이 읽기 좋게 포맷팅해 텔레그램으로 보낸다.

```json
{
  "wakeAgent": true,
  "data": {
    "today": "2026-04-15",
    "calendars": [
      { "name": "개인", "events": [] },
      { "name": "가족", "events": [
        { "summary": "치과 예약", "time": "15:30~16:30", "location": null }
      ]},
      { "name": "회사", "events": [
        { "summary": "주간 스탠드업", "time": "10:00~10:30", "location": "Zoom" }
      ]}
    ]
  }
}
```

## Step 3: NanoClaw 태스크 등록

자연어로 NanoClaw에게 말하면 된다:

> "매일 아침 8시에 구글 캘린더 일정 브리핑해줘"

내부적으로는 `schedule_task`가 호출되며, 3가지를 설정한다:

| 파라미터 | 값 | 설명 |
|---------|---|------|
| `schedule_type` | `cron` | 반복 스케줄 |
| `schedule_value` | `0 8 * * *` | 매일 오전 8시 (로컬 타임존) |
| `context_mode` | `isolated` | 대화 기록 불필요, 독립 실행 |
| `script` | Node.js ICS 파서 | 에이전트 기동 전 데이터 수집 |
| `prompt` | "일정을 정리해서 보내줘" | 에이전트가 수행할 작업 |

### 실행 흐름

```
[매일 08:00]
    │
    ▼
[스크립트 실행] ── fetch(iCal URL ×3) → ICS 파싱 → 오늘 일정 필터
    │
    ▼ { wakeAgent: true, data: { calendars: [...] } }
    │
[에이전트 기동] ── data를 받아 포맷팅
    │
    ▼
[메시지 전송] ── 텔레그램 그룹으로 브리핑 발송
```

### 수정과 관리

캘린더를 추가하거나 시간을 변경하고 싶으면 자연어로 말하면 된다:

- "캘린더 하나 더 추가해줘" → `update_task`로 스크립트의 CALS 배열 수정
- "브리핑 시간 7시로 바꿔줘" → `update_task`로 cron 표현식 수정
- "캘린더 브리핑 꺼줘" → `cancel_task` 또는 `pause_task`

## 한계와 개선 방향

**현재 한계:**
- iCal URL은 읽기 전용 — 일정 추가/수정은 불가
- ICS 파일 갱신에 수분~수십 분 지연 (전날 밤에 추가한 일정은 보통 OK)
- RRULE의 모든 패턴을 지원하지는 않음 (BYMONTHDAY, BYSETPOS 등 복잡한 규칙은 미지원)
- EXDATE(반복 일정의 예외 날짜) 처리 미구현

**개선 아이디어:**
- 오전 브리핑 + 오후 3시에 "남은 일정" 리마인더 추가
- 일정 시작 30분 전 알림 (interval 태스크로 구현 가능)
- Google API 서비스 계정으로 전환하면 일정 생성도 가능
- 여러 사람의 캘린더를 합쳐서 "오늘 누구 만나는지" 요약

## 정리

| 구성 요소 | 역할 |
|-----------|------|
| 구글 캘린더 iCal URL | 인증 없이 .ics 데이터 제공 |
| NanoClaw 태스크 스크립트 | ICS fetch → 파싱 → 오늘 일정 필터링 |
| NanoClaw 에이전트 | 데이터를 사람이 읽기 좋게 포맷팅 |
| 텔레그램 그룹 | 최종 브리핑 수신 |

핵심은 **iCal 비공개 URL**이라는 구글의 기존 기능을 활용한 것이다. OAuth 없이, 서버 없이, 스크립트 하나로 매일 아침 일정 브리핑을 받을 수 있다.
