# OpenClaw 텔레그램 파일 전송 가이드

OpenClaw assistant에게 텔레그램으로 파일(PDF, 이미지, 텍스트 등)을 보내달라고 요청하고 실제로 받는 방법을 단계별로 설명한다.

---

## Step 1. outbound 폴더 만들기

OpenClaw terminal에서 아래 명령어를 실행한다.

```bash
mkdir -p ~/.openclaw/media/outbound
```

이 폴더가 파일 전송의 기본 경로가 된다.

---

## Step 2. AGENTS.md에 파일 전송 규칙 추가하기

OpenClaw 프로젝트의 `AGENTS.md` 파일을 열고, 아래 내용을 그대로 복사해서 붙여넣는다.

```markdown
## Sending files via chat channels (MEDIA directives)

Reply-media normalizer (`reply-media-paths.runtime-B671n_FS.js`) whitelists **absolute paths under these managed roots only**:

- `~/.openclaw/media/outbound/…`
- `~/.openclaw/media/tool-*/…`

Everything else goes through `mediaAccess` policy checks and is usually **silently dropped** (search logs for `dropping blocked reply media`).

Rules by file source:

- **Image/music/video generation tools** (`image.generate` etc.): auto-saved under `~/.openclaw/media/tool-*/`. The tool result already contains a `MEDIA:<absolute-path>` line — just include that line verbatim in your reply. No extra staging.
- **Browser outputs** (`browser.screenshot`, downloads): saved under `~/.openclaw/media/browser/` which is **NOT** whitelisted. Copy the file first:
  `cp <browser-path> ~/.openclaw/media/outbound/<YYYY-MM-DD>/<filename>`
  then reference the new absolute path with `MEDIA:`.
- **`exec`-created files** (PDF, MD, DOCX, arbitrary): write the file directly under `~/.openclaw/media/outbound/<YYYY-MM-DD>/<filename>` and emit:
  `MEDIA:/home/<your-username>/.openclaw/media/outbound/<YYYY-MM-DD>/<filename>`

Hard rules:

- **Use absolute paths** to a managed root. Do not use relative `MEDIA:outbound/…` — relatives are resolved against the workspace dir, not the media dir, and behave differently.
- **Never** use `MEDIA:file://…` — explicitly rejected (`"Host-local MEDIA file URLs are blocked in normal replies."`).
- **Send the `MEDIA:` line bare** — it must be the entire reply, on its own, with no accompanying caption, no leading `[[reply_to_current]]`, no backticks or markdown wrapping. Verified: `MEDIA:/home/<your-username>/.openclaw/media/outbound/…` alone delivers; the same path with a directive tag or prefix text has been observed to silently drop.
- If you need caption text, send the file first as a bare-line reply, then send the caption as a separate follow-up message.
- Namespace filenames with a date or session prefix to avoid collisions.
```

> `<your-username>` 부분은 본인의 Linux 사용자명으로 바꿔야 한다. `whoami` 명령어로 확인할 수 있다.

---

## Step 3. 텔레그램에서 파일 요청하기

설정이 끝나면 텔레그램 채팅에서 assistant에게 파일을 요청할 수 있다.

**예시 — PDF 생성 후 전송 요청:**

```
오늘 회의 내용을 PDF로 정리해서 보내줘
```

```
~/documents/report.pdf 파일을 텔레그램으로 보내줘
```

**예시 — 이미 있는 파일 전송 요청:**

```
~/.openclaw/media/outbound/ 에 있는 파일 목록 보여주고, report.pdf를 텔레그램으로 보내줘
```

assistant가 AGENTS.md 규칙에 따라 파일을 outbound 폴더에 준비한 뒤 텔레그램으로 전송한다.

---

## Step 4. 직접 파일을 넣어서 보내기 (수동)

assistant 요청 대신 직접 파일을 준비할 수도 있다.

```bash
# 날짜별 폴더 만들기
mkdir -p ~/.openclaw/media/outbound/$(date +%Y-%m-%d)

# 보낼 파일 복사
cp ~/documents/report.pdf ~/.openclaw/media/outbound/$(date +%Y-%m-%d)/report.pdf
```

그 다음 텔레그램에서 assistant에게 이렇게 요청한다.

```
~/.openclaw/media/outbound/2026-04-18/report.pdf 를 텔레그램으로 보내줘
```

---

## 주의: 절대 하지 말 것

`openclaw.json`에 아래 설정을 넣으면 **모든 tool이 마비된다.** assistant가 아무 작업도 못 하고 대답만 반복하게 되므로 절대 추가하지 않는다.

```json
"tools": {
  "profile": "coding",
  "allow": ["message"]
}
```

이미 넣었다면 `"allow": ["message"]` 줄을 삭제하고 세션을 재시작한다.

## 트러블슈팅 체크리스트

파일이 텔레그램에 도착하지 않을 때 순서대로 확인한다.

- [ ] `~/.openclaw/media/outbound/` 폴더가 존재하는가?
- [ ] AGENTS.md에 파일 전송 규칙이 추가되어 있는가?
- [ ] AGENTS.md 안의 경로에 본인 사용자명이 정확히 들어가 있는가?
- [ ] `openclaw.json`에 `"allow": ["message"]` 설정이 없는가?
